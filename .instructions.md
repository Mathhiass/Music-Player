This repository contains a Next.js App Router frontend under `music-app/` and a Prisma schema at the repository root. When working in this project, follow these guidelines:

- Treat `music-app/` as the primary UI area and `prisma/` plus root config files as the backend/data area.
- Audit first before making code changes. Identify configuration, dependency, and architecture issues before editing files.
- Avoid rewriting working code unnecessarily. Prefer targeted fixes and use the smallest safe change to resolve issues.
- Preserve existing functionality, especially app routing and data models.
- Do not add backend services or external frameworks unless the user explicitly asks for them.
- Use `next` and `react` versions from `music-app/package.json`, and keep Tailwind v4 configuration in place.
- Keep environment variable handling consistent with Next.js and Prisma expectations.
- If you need a new instruction or helper file, place it in the repository root or under `music-app/` only, not in a separate global user profile.
- When making recommendations, explain the exact file and the minimal code change required.
